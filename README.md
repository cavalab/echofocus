# EchoFocus

EchoFocus is an AI method for echocardiography that diagnoses and measures cardiac function based on videos comprising an echocardiogram study.   
Its name refers to the fact that EchoFocus skips view classification, instead relying on attention mechanisms to determine which echo views to priortize in making specfic predictions. 

# Install

Project dependencies are in `pyproject.toml` and `uv.lock`. 

## Requirements:

- Python >= 3.9
- Cuda-enabled NVIDA GPU 

## Quickstart (uv)

Using [uv](https://docs.astral.sh/uv/):

```bash
uv sync # sync dependencies
uv run echofocus --help
```

## Quickstart (pip)

```bash
python -m venv .venv
source .venv/bin/activate
pip install -e .
```


# Usage

You can run the echofocus CLI by calling `uv run echofocus ...`,  `python -m echofocus`, or simply `echofocus ...`, depending on your env setup.

## Extract video embeddings

EchoFocus works on top of the video embeddings generated by [PanEcho](https://github.com/CarDS-Yale/PanEcho/). 
In order to use EchoFocus, you must first generate video embeddings for your echos, and store them in `hd5` files. 
See the [embed](https://github.com/cavalab/echofocus/tree/main/embed) for scripts and guidelines on how to accomplish this.

From there, you can use EchoFocus to train models, generate study embeddings, generate new predictions, and analyze video importance according to the functions below. 

## Load a pre-trained model

Pre-trained models are made available via the [Releases](https://github.com/cavalab/echofocus/releases) page. 
Once downloaded, they can be used through the `echofocus` CLI as described below. 

## Train Models

To train a model, you must first create a configuration file named `config.json`. 
An example file called `config-example.json` is included as a template for you to use. 
The `config.json` file specifies:
- for each task configuration, and the path to the labels file, and the columns in the labels file to use for training.  
- for each dataset, the the path to the pre-generated PanEcho video embeddings. 

Once `config.json` is in place, you can train a model with `echofocus train` by specifying at minimum a model name, dataset, and task. 

```bash
uv run echofocus train \
  --model_name EchoFocus_CHD_MultiQuery \
  --config config.json \
  --dataset bch_internal \
  --task chd \
  --transformer_type multiquery \
  --end_to_end False \
  --learning_rate 1e-4 \
  --encoder_depth 1
```

- Trained models and training results are stored in `./trained_models/[model_name]`. If the `model_name` you specify already exists, that one will be loaded from the best checkpoint and training will resume according to the specified arguments.
- Task information is stored in `config.json`. 
- The `dataset` is used as a key lookup to `config['dataset'][dataset]` to load the paths to the label files.

Important training notes:
- `transformer_type` currently supports `standard`, `query`, and `multiquery`.
- `end_to_end=False` trains on precomputed PanEcho embeddings.
- `end_to_end=True` trains from raw clips with the PanEcho backbone in the loop.

## Ping-Pong End-to-End Training

For alternating transformer-only / PanEcho-only end-to-end training, use `train_ping_pong`.

```bash
uv run echofocus train_ping_pong \
  --model_name chd_e2e_ping_pong \
  --config config.json \
  --dataset bch_internal \
  --task chd \
  --end_to_end True \
  --num_clips 1 \
  --clip_len 16 \
  --total_epochs 100 \
  --panecho_lr_ratio 0.1 \
  --use_hdf5_index \
  --parallel_processes 4
```

`panecho_lr_ratio` sets the PanEcho phase learning rate relative to `learning_rate`. The transformer phase uses `learning_rate`, and the PanEcho phase uses `learning_rate * panecho_lr_ratio`.

## Evaluate a Trained Model

Use `evaluate` to run a saved checkpoint on one or more folds and emit prediction CSVs plus metric JSONs.

```bash
uv run echofocus evaluate \
  --model_name EchoFocus_CHD \
  --config config.json \
  --dataset bch_internal \
  --task chd \
  --folds "('test',)"
```

This writes artifacts such as:
- `saveout_test_<dataset>.csv`
- `metrics_test_<dataset>.json`
- `runtime_config.json`

## Generate Study Embeddings

To generate study-level embeddings, use `embed`. 
For example:

```bash
uv run echofocus embed \
  --dataset outside \
  --model_name EchoFocus_Measure
```

Would generate study embeddings using the EchoFocus_Measure model on the "outside" dataset.

## Explain Model Outputs

EchoFocus supports model explanations that attribute importance to individual videos in the echo study using integrated gradients. 
This is accomplished by calling `explain` with the appropriate arguments. For example:

```bash
uv run echofocus explain \
  --model_name EchoFocus_CHD \
  --config config.json \
  --dataset bch_internal \
  --task chd \
  --explain_tasks all
```

## Batch Experiment Submission

For Slurm sweep generation and submission, use [submit_jobs.py](/lab-share/CHIP-Lacava-e2/Public/bill/projects/echo-focus/echofocus/submit_jobs.py#L1).

```bash
uv run python submit_jobs.py run_config --config experiments/end2end.yaml --print_only True
uv run python submit_jobs.py run_config --config experiments/end2end.yaml --submit True
```

The sweep config accepts YAML and supports:
- top-level Cartesian-product hyperparameters such as `tasks`, `transformer_types`, `learning_rates`, and `encoder_depths`
- `experiments:` blocks for constrained sub-grids such as coupling specific `clip_lengths` and `num_clips`

See [experiments/end2end.yaml](/lab-share/CHIP-Lacava-e2/Public/bill/projects/echo-focus/echofocus/experiments/end2end.yaml#L1) for a current example.
